# Company Leaderboard Client Application

## Project Overview
Build a **Company Leaderboard Client Application** that displays company members ranked by their performance score.

---

## Core Features

### 1. Leaderboard Display
- Show a list of all company members sorted in **descending order by score** (highest score first).
- Each member entry should include relevant details:
  - Name
  - Score
  - Rank
  - Optional avatar

---

### 2. Top 3 Highlight
- The **top 3 users** should be visually separated and displayed at the top of the page.
- These users should have a distinct UI:
  - Larger cards
  - Badges
  - Podium-style layout

---

### 3. Filter Bar
- Positioned **above the Top 3 section**.
- Should support filtering leaderboard data by:
  - **Year** (e.g., 2023, 2024, 2025)
  - **Quarter** (Q1, Q2, Q3, Q4)
  - **Category** (e.g., Sales, Engineering, Support)

- Filters should be **combinable**:
  - Example: `Year + Quarter + Category`

---

### 4. Search Functionality
- Provide a search input to allow users to **find a specific member by name**.
- Search behavior:
  - Works in real-time or on submit
  - Matching users are highlighted or filtered in the leaderboard

---

## Behavior & UX Requirements
- Filtering and search should update the leaderboard **dynamically**.
- Ensure smooth UX when combining filters and search.
- Handle **empty states** (e.g., no results found).
- Maintain consistent ranking after filters/search are applied.

---

## Technical Expectations
- Clean, modular, and maintainable code.
- Clear separation of concerns:
  - UI
  - State management
  - Data fetching
- Optimized performance for large datasets:
  - Pagination and/or virtualization
